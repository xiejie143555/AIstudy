# 现实约束 real.md

## 产品名称：日子
**slogan**：不要满不在乎地过日子，好像你可以活一千年似的

## 必须遵守的现实约束

### 1. 隐私保护约束
- **用户记录的信息仅本人可见**
  - 所有用户数据默认完全私密，不允许任何形式的共享或公开
  - 系统设计上必须确保即使是开发者或管理员也无法查看用户内容
  - 数据加密存储，用户持有解密密钥

### 2. 数据自由约束
- **数据必须支持随时导出为通用格式（如 Markdown 或 JSON），不依赖特定平台生存**
  - 提供一键导出功能，支持标准格式（Markdown、JSON、CSV等）
  - 导出过程必须简单直观，不需要技术背景
  - 导出的数据应该是人类可读的，不依赖特定软件打开
  - 支持定期自动备份到用户指定的本地或云存储位置

### 3. 使用效率约束
- **从打开产品到开始输入的路径必须最短，且支持离线记录，网络恢复后同步**
  - 启动时间不超过1秒
  - 最多2步操作即可开始记录（打开应用→开始输入）
  - 支持完全离线使用，所有核心功能不依赖网络
  - 网络恢复后自动同步，不丢失任何离线期间的数据
  - 同步过程对用户透明，不需要手动干预

### 4. AI功能约束
- **根据记录的内容生成数字分身，可以与自己对话（为确保用户隐私，必须用户授权，用户可以选择授权哪些内容用于生成数字分身）**
  - 数字分身功能默认关闭，必须用户明确开启
  - 用户可以精确控制哪些内容用于训练数字分身
  - 提供内容级别的授权控制（单条记录、特定标签、时间范围等）
  - 用户可以随时查看、修改或撤销授权
  - 数字分身数据与用户原始记录严格分离存储
  - 提供一键删除数字分身功能

## 技术实现约束

### 性能约束
- 应用响应时间不超过100ms
- 支持至少10年的数据存储（假设每天100条记录）
- 搜索功能在1秒内返回结果

### 兼容性约束
- 支持主流平台（iOS、Android、Web、桌面端）
- 数据格式跨平台兼容
- 支持旧版本数据导入

### 可靠性约束
- 数据丢失率为0（通过本地备份和云同步双重保障）
- 系统崩溃时，正在编辑的内容不丢失
- 提供数据完整性校验机制
